# Design Context: zquant

## Project Overview

**zquant** is a quantitative trading system designed for professional traders who need to monitor markets, configure strategies, analyze data, and manage trading tasks in real-time.

## Target Audience

**Primary Users**: Data Analysts
- Need clear data visualization and analysis tools
- Balance between professional capability and ease of use
- May have varied technical backgrounds
- Value both functionality and usability

## Primary Use Cases

1. **Strategy Configuration**: Setting and adjusting trading strategy parameters
2. **Data Analysis**: Reviewing historical data and analysis results
3. **Task Management**: Managing and scheduling quantitative trading jobs
4. **Real-time Monitoring**: Continuous market and task status tracking

## Brand Personality & Aesthetic Direction

**Style**: Balanced & Practical
- Professional capability with approachable interface
- Dark theme optimized for extended screen time
- Clear information hierarchy without overwhelming density
- Balance between efficiency and learnability
- Status-driven visual feedback

## Design System

**Color Palette**:
- Primary: Teal (#26a69a) - professional, trustworthy
- Background: Deep dark (#0a0a0a) - reduces eye strain
- Status colors: Blue (running), Green (success), Red (error), Gray (idle)

**Typography**:
- System font stack: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto
- Size scale: 11px-18px (compact for information density)

**Spacing**: Tight but breathable (4px-24px scale)

**Components**: Ant Design Vue as foundation + custom primitives

## Design Principles

1. **Clarity over Beauty**: Function first, aesthetics support usability
2. **Information Density**: Maximize data visibility without overwhelming
3. **Status Visibility**: Clear visual feedback for system states
4. **Consistency**: Unified token system across all surfaces
5. **Performance**: Fast, responsive interactions for trading scenarios

## Anti-Patterns (Avoid)

- ❌ Decorative elements that don't serve function
- ❌ Bright, saturated colors that cause eye fatigue
- ❌ Excessive animations or transitions
- ❌ Large spacing that wastes screen real estate
- ❌ Unclear status indicators or ambiguous states
- ❌ Inconsistent use of design tokens
