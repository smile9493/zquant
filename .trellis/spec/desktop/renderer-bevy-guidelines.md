# Bevy 渲染集成规范

## 适用范围

适用于 `crates/renderer-bevy` 与 `ui-workbench` 的中心画布集成。

开始实现前必须先读：`./architecture-boundaries.md`。

## 集成模式

- 采用“壳核分离”：`egui` 负责应用壳，`Bevy` 负责图形渲染。
- 推荐离屏渲染（Offscreen Texture）再嵌入 `egui` 画布区域。
- 禁止让 Bevy 接管整个桌面 UI 生命周期。
- 禁止使用 `bevy_ui` 作为主业务 UI 框架。

## 渲染职责

- 必备能力：K 线、成交量、十字光标、平移缩放。
- 渲染状态输入统一来自 `RenderScene / RenderCommand`，不得直接读取 UI 临时状态。
- 每帧渲染必须可在无新数据时稳定降载（避免无意义高频重绘）。
- 渲染系统不得承载业务用例与持久化决策，ECS 不得作为业务数据库。

## 数据与状态同步

- UI → Render：通过命令与快照下发，不直接共享可变引用。
- Render → UI：通过事件回传交互结果（光标位置、选中区间等）。
- 禁止在渲染线程中直接访问数据库或网络。
- 渲染派生状态（相机、纹理句柄、帧统计、hover/picking）仅在渲染层维护。
- 业务真状态（策略、任务、数据源配置）不得落入渲染层 resource。

## 性能约束

- 明确帧率目标与降级策略（如重计算节流、视窗裁剪）。
- 大数据量下优先做采样与窗口化，不在 UI 主线程做全量转换。
- 渲染异常不得拖垮主应用进程，需可恢复并记录诊断信息。

## 验收要点

- 中心画布可嵌入且稳定刷新。
- 缩放与平移无明显卡顿或跳变。
- UI 与 Render 的状态同步路径可追踪、可调试。
- 输入焦点冲突经 `app_shell` 路由后行为一致（滚轮、拖拽、快捷键不抢占）。
