# Windows 运行规范

## 适用范围

适用于 zquant 桌面版在 Windows 平台的运行、安装、诊断与目录治理。

## 目录约定

建议使用以下目录：

- `%APPDATA%\\zquant\\config`
- `%LOCALAPPDATA%\\zquant\\logs`
- `%LOCALAPPDATA%\\zquant\\data\\parquet`
- `%LOCALAPPDATA%\\zquant\\tmp`

要求：

- 禁止把可变运行数据写入程序安装目录。
- 日志、数据、临时文件必须物理隔离。

## 连接与依赖

- PostgreSQL 优先 `localhost TCP` 连接，不依赖隐式本地 socket。
- 启动前执行最小依赖自检：数据库连通、目录可写、磁盘空间、版本兼容。
- 自检失败时提供可执行修复建议（而非仅错误码）。

## 日志与诊断

- 启动日志必须包含版本、配置来源、关键依赖状态。
- 错误日志应含上下文（阶段、资源标识、失败原因、重试信息）。
- 支持导出诊断包（日志 + 配置摘要 + 环境摘要，脱敏后）。

## 安装与升级

- Phase 1 可使用内部安装包或便携方案；企业版建议 MSI/EXE 标准安装。
- 升级前后需校验配置兼容与数据目录可访问性。
- 升级失败必须可回滚到上一可用版本。

## 安全基线

- 敏感配置不得明文散落在多个目录。
- 关键目录权限遵循最小权限原则。
- 预留证书、签名、升级包校验的扩展位。

## 验收要点

- 首次安装后可完成一键启动与基础自检。
- 异常退出后日志完整、可用于定位。
- 常见环境问题（DB 不可达、目录不可写）有明确提示与指引。
